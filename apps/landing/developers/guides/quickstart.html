Quickstart Guide

From zero to your first API call in 5 minutes
Step 1 — Create your pixdrift account

Create your account

If you don't have a pixdrift account, sign up at app.bc.pixdrift.com. You'll create your organization during onboarding.

Note your Organization ID from Admin → Settings — you'll need it for some API calls.

Step 2 — Get your API key

Create an API key

Navigate to Admin → Settings → API Keys in the pixdrift app, or create one via the API:

curl -X POST https://api.bc.pixdrift.com/api/admin/api-keys \
  -H "Authorization: Bearer YOUR_SUPABASE_JWT" \
  -H "Content-Type: application/json" \
  -d '{"name": "My First Integration", "scopes": ["read", "write"]}'

You'll receive a key like pd_live_a1b2c3d4e5f6...

Store your key securely — it's only shown once. Use environment variables, not hardcoded strings.

Set your key as an environment variable

# Add to your .env or shell profile
export PIXDRIFT_API_KEY=pd_live_your_key_here
Step 3 — Make your first API call

Verify with the admin dashboard

The admin dashboard is the perfect first call — it returns a snapshot of your organization.

curl https://api.bc.pixdrift.com/api/dashboards/admin \
  -H "Authorization: Bearer $PIXDRIFT_API_KEY"

You should see something like:

{
  "total_users": 12,
  "total_deals": 34,
  "total_revenue": 485000,
  "active_processes": 7,
  "open_nc": 2
}
✅ If you got a response, your API key is working!
Step 4 — Create your first deal

Create a deal via API

Let's add a deal to the pipeline:

curl -X POST https://api.bc.pixdrift.com/api/deals \
  -H "Authorization: Bearer $PIXDRIFT_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Acme Corp Q2 License",
    "value": 125000,
    "currency": "SEK",
    "status": "proposal",
    "expected_close_date": "2026-06-30"
  }'

You'll get back the full deal object with its new UUID. Save the id for the next step.

Move the deal forward

curl -X PATCH https://api.bc.pixdrift.com/api/deals/DEAL_ID/status \
  -H "Authorization: Bearer $PIXDRIFT_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"status": "closed_won"}'

Same pattern in JavaScript

const API = 'https://api.bc.pixdrift.com'
const headers = {
  'Authorization': `Bearer ${process.env.PIXDRIFT_API_KEY}`,
  'Content-Type': 'application/json'
}

// Create deal
const deal = await fetch(`${API}/api/deals`, {
  method: 'POST',
  headers,
  body: JSON.stringify({
    title: 'Acme Corp Q2 License',
    value: 125000,
    currency: 'SEK',
    status: 'proposal'
  })
}).then(r => r.json())

console.log('Created deal:', deal.id)

// Update status
await fetch(`${API}/api/deals/${deal.id}/status`, {
  method: 'PATCH', headers,
  body: JSON.stringify({ status: 'closed_won' })
})
Step 5 — Set up a webhook

Get real-time events

Instead of polling the API, subscribe to events. When a deal closes, you'll get an instant notification.

1. Expose a public endpoint

For local development, use ngrok to expose your localhost:

ngrok http 3000
# → https://abc123.ngrok.io

2. Handle the webhook

const express = require('express')
const app = express()

app.post('/webhook', express.json(), (req, res) => {
  const { type, data } = req.body

  if (type === 'deal.won') {
    console.log(`🎉 Deal won: ${data.title} — ${data.value} SEK`)
    // Trigger Slack notification, CRM update, etc.
  }

  res.json({ received: true })
})

app.listen(3000)

3. Register your webhook URL

Contact API support or use the pixdrift admin UI to register your webhook endpoint. Full webhook docs at webhooks documentation.
