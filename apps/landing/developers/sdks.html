SDKs & Client Libraries

Official SDKs make it easier to integrate with pixdrift in your preferred language. All SDKs are wrappers around the REST API — you can always use the raw API directly.

SDKs are currently in development. The REST API is available now and works with any HTTP client. Use the Quickstart Guide to get started today.
🟢

Node.js / TypeScript SDK Coming Soon

Full TypeScript support with auto-complete for all endpoints and response types.

Installation

npm install @pixdrift/sdk
# or
yarn add @pixdrift/sdk
# or
pnpm add @pixdrift/sdk

Quick Example

import { PixdriftClient } from '@pixdrift/sdk'

const client = new PixdriftClient({
  apiKey: process.env.PIXDRIFT_API_KEY,
  // baseUrl: 'https://api.bc.pixdrift.com' (default)
})

// List contacts
const contacts = await client.contacts.list()

// Create a deal
const deal = await client.deals.create({
  title: 'Enterprise Q2',
  value: 250000,
  currency: 'SEK',
  status: 'proposal'
})

// Update deal status
await client.deals.updateStatus(deal.id, 'closed_won')

// Get admin dashboard
const dashboard = await client.dashboards.admin()

// Generate income statement
const report = await client.reports.incomeStatement({
  from: '2026-01-01',
  to: '2026-03-31'
})

TypeScript Types

import type { Contact, Deal, Task, CapabilityProfile } from '@pixdrift/sdk'

Go SDK Coming Soon

Idiomatic Go client with context support and structured error handling.

Installation

go get github.com/pixdrift/go-sdk

Quick Example

package main

import (
    "context"
    "fmt"
    pixdrift "github.com/pixdrift/go-sdk"
)

func main() {
    client := pixdrift.NewClient(pixdrift.Config{
        APIKey: "pd_live_...",
    })

    ctx := context.Background()

    // List contacts
    contacts, err := client.Contacts.List(ctx)
    if err != nil {
        panic(err)
    }
    fmt.Printf("Found %d contacts\n", len(contacts))

    // Create deal
    deal, err := client.Deals.Create(ctx, pixdrift.DealInput{
        Title:    "Enterprise Q2",
        Value:    250000,
        Currency: "SEK",
    })
    if err != nil {
        panic(err)
    }
    fmt.Println("Deal created:", deal.ID)
}

Generate Your Own SDK

Use the OpenAPI spec to generate a client in any language.

The pixdrift OpenAPI 3.0 spec is available and can be used to auto-generate clients in 40+ languages:

# Download the spec
curl -o pixdrift.yaml https://api.bc.pixdrift.com/openapi.yaml

# Generate a client with openapi-generator
openapi-generator-cli generate \
  -i pixdrift.yaml \
  -g typescript-fetch \
  -o ./pixdrift-client

# Or with fern
fern generate --api pixdrift.yaml --language typescript

Supported generators: typescript, typescript-fetch, typescript-axios, csharp, java, kotlin, ruby, rust, php, swift, and more.

Using the REST API directly

You don't need an SDK. The API works with any HTTP client. Here's how to call it in common environments:

Browser (fetch)

const apiKey = 'pd_live_...'
const base = 'https://api.bc.pixdrift.com'
const headers = { 'Authorization': `Bearer ${apiKey}`, 'Content-Type': 'application/json' }

const contacts = await fetch(`${base}/api/contacts`, { headers }).then(r => r.json())

Server-side (Node.js — native fetch or axios)

// Using axios
const axios = require('axios')
const client = axios.create({
  baseURL: 'https://api.bc.pixdrift.com',
  headers: { 'Authorization': `Bearer ${process.env.PIXDRIFT_API_KEY}` }
})
const { data: contacts } = await client.get('/api/contacts')
