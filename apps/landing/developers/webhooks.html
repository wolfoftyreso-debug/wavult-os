Webhooks

pixdrift sends real-time webhook events to your endpoints when important things happen in the system. Build event-driven integrations without polling the API.

Webhooks are sent as HTTP POST requests with a JSON body and an HMAC-SHA256 signature header for security verification.

Event Types

All events pixdrift can send to your webhook endpoint:

Deals

deal.created
A new deal was added to the pipeline
deal.updated
Deal properties were modified
deal.won
Deal moved to closed_won
deal.lost
Deal moved to closed_lost

Tasks

task.created
A new task was created
task.completed
Task marked as completed
task.overdue
Task passed its due date without completion

Invoices

invoice.created
New invoice generated
invoice.paid
Invoice payment registered

Processes

process.started
Process execution started
process.completed
All process steps completed

Non-Conformance

non_conformance.opened
New NC report filed
non_conformance.closed
NC report closed/resolved

Users

user.invited
User invitation sent
user.activated
User completed registration

Payload Format

Every webhook has a consistent envelope structure:

{
  "id": "evt_01HX9...",           // Unique event ID (idempotency)
  "type": "deal.won",              // Event type
  "timestamp": "2026-03-21T10:00:00Z",
  "org_id": "uuid",                // Your organization
  "data": {
    // The full resource object that triggered the event
    "id": "uuid",
    "title": "Enterprise License Q1",
    "value": 250000,
    "status": "closed_won",
    // ... full resource fields
  }
}

Signature Verification

Always verify webhook signatures. pixdrift signs every payload with HMAC-SHA256 using your webhook secret.

Header

X-Pixdrift-Signature: sha256=a1b2c3d4...

Node.js / Express

const express = require('express')
const crypto = require('crypto')

const app = express()

// IMPORTANT: Use raw body (not parsed JSON) for signature verification
app.post('/webhook', express.raw({ type: 'application/json' }), (req, res) => {
  const signature = req.headers['x-pixdrift-signature']
  const secret = process.env.PIXDRIFT_WEBHOOK_SECRET

  // Compute expected signature
  const expected = 'sha256=' + crypto
    .createHmac('sha256', secret)
    .update(req.body)
    .digest('hex')

  // Timing-safe comparison (prevents timing attacks)
  const valid = crypto.timingSafeEqual(
    Buffer.from(expected, 'utf8'),
    Buffer.from(signature, 'utf8')
  )

  if (!valid) {
    return res.status(401).json({ error: 'Invalid signature' })
  }

  const event = JSON.parse(req.body)
  
  // Handle event types
  switch (event.type) {
    case 'deal.won':
      console.log('🎉 Deal won!', event.data.title, event.data.value)
      break
    case 'task.overdue':
      // Send Slack notification, etc.
      break
  }

  res.status(200).json({ received: true })
})

app.listen(3000)

Python / Flask

import hmac, hashlib
from flask import Flask, request

app = Flask(__name__)

@app.route('/webhook', methods=['POST'])
def webhook():
    signature = request.headers.get('X-Pixdrift-Signature', '')
    secret = os.environ['PIXDRIFT_WEBHOOK_SECRET']
    
    expected = 'sha256=' + hmac.new(
        secret.encode(), request.data, hashlib.sha256
    ).hexdigest()
    
    if not hmac.compare_digest(expected, signature):
        return {'error': 'Invalid signature'}, 401
    
    event = request.json
    print(f"Event: {event['type']}")
    
    return {'received': True}

Retry Logic

If your endpoint doesn't respond with 2xx within 30 seconds, pixdrift will retry the delivery.

Attempt 1
Immediate delivery
Attempt 2
5 minutes after failure
Attempt 3
30 minutes after failure

After 3 failed attempts, the event is marked as failed. Each event has a unique id — use this for idempotency to avoid processing the same event twice on retries.

Best practice: Return 200 OK immediately and process the event asynchronously (e.g., add to a queue). Don't make your webhook handler do heavy work synchronously.
