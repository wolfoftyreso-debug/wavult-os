Quick Start
Up and running in 3 steps
From zero to your first API call in minutes.
1

Get your API key

In the pixdrift app, go to Admin → Settings → API Keys and create your first key.

pd_live_<your_key_here>
2

Make your first request

Hit the admin dashboard endpoint to verify everything works.

curl -H "Authorization: Bearer pd_live_..." \ https://api.bc.pixdrift.com/api/dashboards/admin
3

Set up a webhook

Register your endpoint to receive real-time events when deals close, tasks complete, and more.

POST /api/webhooks { "url": "https://your.app/hook" }

Rate Limits
Fair use policy

Rate limits are per API key, tracked in a sliding 15-minute window. Response headers tell you exactly where you stand.

Plan Requests / 15 min Requests / day Burst
Free 50 500 10
Standard 100 5,000 30
Growth 300 20,000 100
Enterprise 1,000 Unlimited Unlimited

Response headers: X-RateLimit-Limit · X-RateLimit-Remaining · X-RateLimit-Reset (Unix timestamp). Status 429 Too Many Requests when exceeded.

Authentication
Simple Bearer token auth

API Key (recommended)

Create long-lived API keys with scoped permissions. Best for integrations.

# All API requests
curl https://api.bc.pixdrift.com/api/contacts \
  -H "Authorization: Bearer pd_live_abc123..."

JWT Token

For user-facing applications. Obtain via Supabase auth, expires in 1 hour.

# User-context requests
curl https://api.bc.pixdrift.com/api/contacts \
  -H "Authorization: Bearer eyJhb..."
