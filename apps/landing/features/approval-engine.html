The problem it solves

Core features

What Approval Engine does.

📹

1-Tap Video Capture, Auto-Linked to Work Order

From the work order screen, the technician taps "Request Approval." The camera opens. They record up to 90 seconds showing exactly what they found — the worn brake pad, the cracked belt, the rust on the undercarriage. The video links to the work order automatically. No manual entry, no searching for the right job.

The technician adds a brief technical note: "Front brake pads at 2mm, recommended minimum 3mm. Needs replacement before next service." That's all they need to provide.

Technical

Video is captured via the device camera API and uploaded via chunked multipart to object storage. Processing pipeline: transcode to H.264 MP4 (720p, max 10MB) → content-addressed storage → linked to work_order_id via approval_request entity. Capture-to-upload time: <30s on 4G.

🤖

AI Translation: Technical Language → Plain Customer Language

The technician's note goes through pixdrift's AI translation layer. Out comes something a customer can actually understand: "Your front brake pads are worn below the safe minimum. Replacing them now takes about 45 minutes and prevents brake failure. We recommend this before returning the car to you."

The AI also generates a clear explanation of consequences: what happens if approved (work is done today), what happens if declined (car returned with documented risk). This reduces customer anxiety and approval hesitation.

The technician reviews the AI output before it's sent. One tap to send if it looks right, or they can edit. Average review time: under 15 seconds.

Technical

Translation uses a fine-tuned language model prompted with: technician note, vehicle context (make, model, year, mileage), workshop's labor rate, and parts pricing from DMS inventory. Output is validated for: readability score, price accuracy against parts cost, and completion of required legal disclosures. Runs in <5 seconds.

📱

SMS with Tokenized Link — No Login for Customer

The customer receives an SMS: "Hi Anna, Lindqvists found something during your service. Watch a short video from the technician and let us know how to proceed: [link]"

The link opens a mobile-optimized page. No login, no app, no account. The token in the URL is cryptographically tied to this specific approval request. It expires after 72 hours or immediately on decision.

Technical

Approval tokens are UUID v4 values HMAC-signed with a per-organization secret. Token expiry is enforced server-side. The approval page URL pattern is /a/{token} and resolves to the workshop's white-label domain if configured. SMS delivery via Twilio with automatic fallback to backup provider on failure.

☑️

Three Choices. Nothing More.

Approve. Reject. Call me. That's it. Customers aren't asked to navigate a portal, fill a form, or make complex decisions. The simplicity is intentional — fewer options means faster decisions, higher approval rates, and less customer friction.

"Call me" logs the request and notifies the service advisor immediately that the customer wants to discuss. It's not a dead end — it's a tracked handoff to a human.

Technical

Each decision is recorded as an approval_response entity with: decision (APPROVED|REJECTED|CALL_REQUESTED), timestamp, customer_ip, user_agent, token_hash, and a SHA-256 hash of the entire approval content shown. This constitutes the legal record of consent.

⚡

Automatic Work Order Advancement on Approval

When the customer approves, the work order in DMS advances to the next stage automatically. The technician receives a push notification on their device. If they've moved to another job, they get a clear queue showing the approved job is ready to proceed.

No service advisor needs to relay the message. No sticky note on the windshield. The system closes the loop.

Technical

Approval webhook fires a approval.approved event that triggers a work order stage transition via the DMS module API. Stage advancement is configurable per workshop — some advance to "In progress," others to "Awaiting parts." Push notifications delivered via APNs/FCM with fallback to SMS if app not installed.

🏷️

White-Label Per Workshop

Every workshop using Approval Engine gets their own branded approval experience. The customer sees the workshop's logo, name, and color scheme. The URL can be the workshop's own domain. pixdrift is invisible to the customer.

This matters for trust. A customer who receives a link that looks like their trusted workshop's website is far more likely to engage than one that goes to an unfamiliar third-party service.

Technical

White-label configuration: custom domain (CNAME to pixdrift CDN), logo upload (PNG/SVG, served from CDN), primary color token, workshop name and contact details. All approval pages are rendered server-side with workshop theme applied. Custom domain SSL is provisioned automatically via Let's Encrypt.

Module integrations

How Approval Engine connects to everything else.

Approval: Approved → DMS Work Order Work order advances to next stage automatically. Technician notified. Approved items added to final invoice line items.
Approval: Rejected → DMS + Process Rejection logged on work order with customer's decision. If a safety item is rejected, a risk record is created in Process with the customer's signed rejection as evidence.
Approval Sent → Execution Task A follow-up task is created for the service advisor if no response received within configured time (default: 2 hours).
Work Order Complete → Finance Invoice Completed and approved items roll up to the final invoice automatically. Approval record is attached to the invoice as documentation.
Call Requested → Execution · Channel Opens a channel thread on the work order flagged for the service advisor, with the approval request linked and customer contact info visible.
API Reference

Build on Approval Engine.

Endpoint Method Description
/v1/approvals POST Create approval request. Required: work_order_id, video_asset_id, tech_note, price_estimate, customer_phone.
/v1/approvals/{id} GET Get approval request status. Returns state: PENDING | APPROVED | REJECTED | CALL_REQUESTED | EXPIRED.
/v1/approvals/{id}/translate POST Trigger AI translation of tech note. Returns customer-facing text for review before sending. Optional language param for non-Swedish customers.
/v1/approvals/{id}/send POST Send approval SMS. Requires prior /translate call. Emits approval.sent webhook. Idempotent — subsequent calls return existing request.
/v1/approvals/{id}/audit GET Full audit record for this approval: content shown, token hash, customer decision, timestamp, IP. Returns signed PDF when format=pdf.
/v1/assets/upload POST Upload video for approval request. Multipart. Max 50MB source video — transcoded to 10MB MP4 by pipeline. Returns asset_id.
